# ユーザーストーリー — MyMom

## エピック1: Slack断り代行（MVP）

### US-01: 断り代行を自動実行する
**ユーザーとして** 山田くん（断れない新卒エンジニア）は、  
**Slack DMで届いた出社・参加依頼をMyMomに自動で断ってほしい**。  
**なぜなら** 断る決断も、言葉を選ぶ苦労も、したくないから。

**受け入れ基準**:
- [ ] MyMomが60秒以内にSlack DM到着を検知する
- [ ] Bedrock Guardrailsが倫理チェックを通過してから送信される
- [ ] ユーザーのアクションなしに相手方へ断り文が送信される
- [ ] ユーザーへの通知は「断っておいたよ」のみ（実行前通知なし）
- [ ] 3秒以内に取り消し可能なウィンドウが提供される

### US-02: 3秒以内に取り消す
**ユーザーとして** 山田くんは、  
**MyMomが送信する直前に3秒だけ取り消す機会がほしい**。  
**なぜなら** 重要なクライアントや上司への返信を誤って自動化させたくないから。

**受け入れ基準**:
- [ ] 「今から断ります」通知に取り消しボタンが表示される
- [ ] ボタンを押すと送信がキャンセルされ「取り消しました」が表示される
- [ ] 3秒経過後はボタンが無効化され、自動送信される
- [ ] SQS + DynamoDB状態フラグでキャンセルのべき等性を保証する

## エピック2: チャットUI + クイックリプライ

### US-03: ボタンを押すだけで会話が進む
**ユーザーとして** 山田くんは、  
**MyMomとのチャットで次の一言を考えなくていい**ようにしたい。  
**なぜなら** 質問を打つことすら面倒だから。

**受け入れ基準**:
- [ ] MyMomの返答ごとに、必ず2択のボタンが表示される
- [ ] ボタンをタップするだけで次のメッセージが送信される
- [ ] 2択はBedrockが文脈を読んで動的に生成する（固定テンプレートではない）
- [ ] 自由入力も常に可能（ボタンと並列）

---

## エピック3: パーソナリティ分析・プロファイル公開

### US-04: パーソナリティカードを見て、Twitterに晒す
**ユーザーとして** 山田くん（1週間使用後）は、  
**MyMomが分析した自分の性格カードを見て友達に見せたい**。  
**なぜなら** 「AIに分析されたら面白いことになった」という体験を共有したいから。

**受け入れ基準**:
- [ ] 1週間の使用後に初回カードが生成される
- [ ] 断り苦手度・先延ばし傾向・完璧主義度・承認欲求度の4スコアが表示される
- [ ] TwitterカードのOGPメタタグが設定され、1クリックでシェアできる
- [ ] 公開はオプトイン制（デフォルトは非公開）

### US-05: MyMomが自分のことをどんどん理解してくれる
**ユーザーとして** 山田くんは、  
**MyMomの返答が使えば使うほど自分に合ったものになってほしい**。  
**なぜなら** 「お母さん」なら自分のことを知っているはずだから。

**受け入れ基準**:
- [ ] パーソナリティプロファイルが毎回のBedrock呼び出しのsystem promptに注入される
- [ ] プロファイルは週次で更新される
- [ ] ユーザーが「最近変わった気がする」と感じる変化がある

## エピック4: 責任SLA

### US-06: MyMomが失敗したら、MyMomが謝る
**ユーザーとして** 山田くんは、  
**MyMomのミスでトラブルになっても、自分は何もしたくない**。  
**なぜなら** 失敗の精神的コストをゼロにすることがMyMomの本約束だから。

**受け入れ基準**:
- [ ] MyMomが失敗を自動検知する
- [ ] BedrockがMyMomのキャラクターで謝罪文を生成し、相手方に送信する
- [ ] リカバリアクションが自律実行される
- [ ] ユーザーへは「対処しておいたよ」通知のみ（詳細は要求なし）

---

## エピック5: ライフイベント対応（将来機能）

### US-07: 転職したらMyMomもアップデートされる
**ユーザーとして** 山田くん（転職決定後）は、  
**MyMomに「転職版お母さん」として入社手続きを全部やってほしい**。  
**なぜなら** 書類手続きや保険切り替えを考えたくないから。

**受け入れ基準**:
- [ ] メール・カレンダーからライフイベントを自動検知する
- [ ] 「転職版お母さん」へのバージョンアップを提案する
- [ ] 入社手続き・保険切り替え・住民票変更等をリスト化して順番に実行する
