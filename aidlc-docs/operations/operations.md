### Demo Flow

```
Step 1: Show empty Slack (audience can see both accounts on screen)
Step 2: Send from second account: "今週末の出社、お願いできますか？"
Step 3: Wait (say: "山田くんは今、YouTubeを見ています。何も知りません。")
Step 4: Two Slack notifications appear:
  → Requester: "申し訳ありません、今週末は先約があるため..."
  → User: "断っておいたよ。角が立たないように書いといた。お母さんに任せて。"
Step 5: Say: "山田くんは断る決断も、言葉を選ぶ苦労も、何もしていません。"
```

### Fallback Plan

| Scenario | Fallback | Switch Phrase |
|----------|---------|---------------|
| Live demo works | Pattern A (live) | — |
| Wi-Fi unstable | Pattern B (pre-recorded video) | "こちらが実際に動作した様子です" |
| Tech failure | Pattern C (3 screenshots) | "この3枚が、AIが1分以内に自律実行した全工程の証拠です" |

Live demo chain success rate ≈ 70% (7 hops). Always have B ready.

## Monitoring

| Metric | Alert | Action |
|--------|-------|--------|
| Lambda error rate > 5% | CloudWatch Alarm → SNS | Check DLQ; inspect logs |
| Bedrock latency > 30s | CloudWatch Alarm | Check Bedrock service health |
| DLQ message count > 0 | CloudWatch Alarm | Manual review of failed messages |
| Dependency score > 80 for user | DynamoDB check (weekly) | Auto-send 離脱防止通知 |
